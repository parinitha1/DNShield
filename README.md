# DNS Shield 🛡️
**Real-time DNS Intrusion Detection System**

An undergraduate research implementation of a Network Intrusion Detection System (NIDS) that classifies DNS queries using unsupervised machine learning.

---

## Architecture

```
Live DNS Traffic (UDP/53)
        │
   [sniffer.py]   ← Scapy packet capture (daemon thread)
        │
   blocker.py     ← Fast-path: already blocked? DROP.
        │
   features.py    ← Extract (length, Shannon entropy)
        │
   dns_model.pkl  ← IsolationForest.predict()
        │
   +1 = clean     -1 = anomaly
        │               │
   log to MongoDB  block + log to MongoDB
        │
   [main.py]      ← FastAPI REST API
        │
   /              ← HTML Dashboard (templates/index.html)
   /logs          ← Query log JSON
   /blocked       ← Block list JSON
   /stats         ← Live counters JSON
```

## Quick Start

### 1. Install dependencies
```bash
pip install -r requirements.txt
```

### 2. Generate training data
```bash
python data_generator.py
# Output: dns_training_data.csv  (2500 rows: 2000 benign + 500 malicious)
```

### 3. Train the model
```bash
python train.py
# Output: dns_model.pkl
# Prints: classification report + confusion matrix
```

### 4. Start MongoDB (Docker)
```bash
docker run -d --name mongo -p 27017:27017 mongo:7
```

### 5. Launch the server (requires root for raw sockets)
```bash
sudo uvicorn main:app --host 0.0.0.0 --port 8000
```

### 6. Open the dashboard
Navigate to **http://localhost:8000**

## Feature Engineering

| Feature | Benign | Malicious (DNS Tunnel) |
|---------|--------|----------------------|
| **Length** | 10–30 chars | 70–120 chars |
| **Shannon Entropy** | 2.1–3.2 bits | 4.5–5.8 bits |

Shannon entropy formula:  
**H(d) = −Σ p(c) · log₂ p(c)**

DNS tunneling encodes binary payloads (Base64/Hex) into subdomains, producing strings that are abnormally long and have near-maximum character diversity (high entropy).

## API Reference

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/` | HTML dashboard |

## Security Notes

- Packet capture requires **root privileges** (or `CAP_NET_RAW` capability).
- The `/unblock` endpoint has no authentication — add an API key middleware for production.
- The IsolationForest is **unsupervised**; it detects anomalies relative to training data. Retrain periodically as network patterns evolve.
